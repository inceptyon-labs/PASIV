# De-Vibe

Make a project stop looking machine-generated. Find the tells AI leaves behind — purple prose, smart punctuation, redundant comments, tracked agent-config files, AI commit trailers — then report them by category and apply the fixes after one confirmation gate.

**Input:** $ARGUMENTS — optional flags:
- `--report` — scan and report only, change nothing
- `--prose` / `--code` / `--gitignore` / `--git` — limit to one category (default: all four)
- `--path <dir>` — scope to a subtree

**Default flow:** scan → categorized report → confirm → apply. Never apply before showing the report.

---

## Phase 1: Preflight

```bash
pwd && git rev-parse --is-inside-work-tree 2>/dev/null
```

If not a git repo, the `gitignore` and `git` categories are skipped — say so and continue with `prose` and `code` only.

Identify what's tracked vs ignored so you never propose editing generated or vendored files:

```bash
git ls-files 2>/dev/null | head -2000
```

Build an exclusion set from the start. **Never scan or modify:** `node_modules/`, `vendor/`, `dist/`, `build/`, `.next/`, `target/`, `*.min.*`, lockfiles, anything in `.gitignore`, anything matching a generated-file header (`@generated`, `Code generated by`, "DO NOT EDIT").

---

## Phase 2: Scan (gather everything before reporting)

Run categories in parallel where possible. Collect counts + file:line locations. Do **not** edit yet.

### Prose (public-facing docs)

Target: `README*`, `docs/**`, `CONTRIBUTING*`, `*.md` that ships to users, site copy, package descriptions. **Skip** internal agent files (`CLAUDE.md`, `AGENTS.md`) — those get gitignored, not rewritten.

Inside markdown, **never touch fenced code blocks or inline `code`** — em-dashes and "utilize" in a code sample are real.

Find:
- **Smart punctuation:** em-dash `—`, en-dash `–` used as em-dash, curly quotes `" " ' '`, ellipsis char `…`, non-breaking space (U+00A0)
- **Filler / LLM tics:** "it's worth noting", "it's important to note", "it is worth", "delve", "leverage", "utilize", "robust", "comprehensive", "seamless(ly)", "elevate", "boasts", "in today's world", "in the world of", "let's dive in", "dive into", "in conclusion", "furthermore", "moreover", "that being said", "navigating the", "at the end of the day", "game-changer", "unlock", "supercharge", "effortless"
- **Rule-of-three cadence:** "X, Y, and Z" adjective triplets in marketing position (flag, don't auto-rewrite — needs judgment)
- **Emoji in headings** and decorative emoji in prose
- **Over-formatting:** bold-label bullet lists (`- **Thing:** ...`) used where flowing sentences belong; suspiciously symmetric section trees

```bash
grep -rnE $'\xe2\x80\x94|\xe2\x80\xa6' --include='*.md' . 2>/dev/null | grep -v node_modules | head -100   # em-dash, ellipsis (-E: BSD grep has no \| alternation)
grep -rn $'\xc2\xa0' --include='*.md' . 2>/dev/null | grep -v node_modules | head -50   # NBSP (UTF-8 C2 A0)
grep -rniE 'delve|leverage|utiliz|seamless|robust|comprehensive|streamline|cutting.?edge|state.?of.?the.?art|next.?generation|best practices|it'"'"'s worth noting|dive in|in conclusion|furthermore|moreover|game.?changer|supercharge' --include='*.md' . 2>/dev/null | grep -v node_modules | head -100
find . -name '*.md' -not -path '*/node_modules/*' -exec perl -CSD -ne 'print "$ARGV:$.:$_" if /[\x{1F300}-\x{1FAFF}\x{2600}-\x{27BF}]/' {} + 2>/dev/null | head -50   # emoji in docs (perl: BSD grep has no -P)
```

### Code comments

Read tracked source files (skip exclusion set). Classify each comment — this is judgment, not regex.

**Propose removal:**
- Restates the code: `// increment counter` over `counter++`, `// import React`, `// loop through items`, `# return the result`
- Banner / divider comments: `// ============`, `// ---- Section ----`, `/* SECTION */`
- Docstrings that only echo the signature with no added information
- Narration of obvious control flow

**Always keep (never touch):**
- *Why* comments — rationale, trade-offs, workarounds, "this looks wrong but…", links to issues/RFCs
- Non-obvious logic, gotchas, units, invariants
- Tool directives: `eslint-disable`, `# type: ignore`, `// @ts-`, `# noqa`, `#pragma`, `// nolint`, shebangs
- License / copyright headers
- Genuine public-API doc comments that document behavior beyond the signature

When unsure, **keep it** and don't list it. Bias toward leaving comments alone.

### Gitignore (AI artifacts)

Find AI/agent config that is **tracked** (would publish to others) — these get gitignored and untracked locally, not deleted:

```bash
git ls-files 2>/dev/null | grep -iE '^(CLAUDE\.md|AGENTS\.md|GEMINI\.md|CONVENTIONS\.md|\.cursorrules|\.aider|\.windsurf|\.continue|\.specstory|\.mcp\.json|\.claude/|\.gemini/|\.codex/|\.cursor/|\.github/copilot-instructions\.md)'
```

Also check whether `.gitignore` already covers each. Report only what's tracked or not-yet-ignored.

> If the project is itself an AI tool (e.g. PASIV) where these files are the product, flag it and ask before ignoring — don't blindly hide a repo's own deliverables.

### Git trailers

```bash
# Full message body (%B) — trailers live in the body, not the subject (%s)
git log --format='%H %B' -n 300 2>/dev/null | grep -iE '🤖|generated with claude|co-authored-by:[[:space:]]*claude' | head -100
# Count of affected commits
git log --grep='Co-Authored-By: Claude' --grep='Generated with Claude' -i --oneline -n 300 2>/dev/null | wc -l
```

Count commits carrying `Co-Authored-By: Claude` / `🤖 Generated with Claude Code` and emoji-prefixed subjects.

---

## Phase 3: Report

Print one compact, categorized report. Counts up top, locations available on request.

```
de-vibe scan — <project>
─────────────────────────────────────
PROSE      <n> em-dashes, <n> smart-quotes, <n> filler  →  README.md, docs/
CODE       <n> restate comments, <n> banner comments    →  src/…
GITIGNORE  <files> tracked, not ignored
GIT        <n> commits with AI trailers
─────────────────────────────────────
Apply which? [prose / code / gitignore / git / all / none]
```

If `--report`, stop here.

Otherwise ask with **AskUserQuestion** which categories to apply (multiSelect). For `git`, surface the history-rewrite sub-choice (Phase 4) only if they pick it.

---

## Phase 4: Apply

Only the confirmed categories. Show a diff summary after each.

### Prose
Rewrite for meaning, don't blind-replace. Em-dash → comma, parentheses, period, or "to"/"is" depending on the sentence. Smart quotes → straight. Ellipsis char → `...`. NBSP → space. Delete filler phrases and tighten the sentence; convert bold-label bullets to sentences where it reads better. Preserve the author's facts and structure — you're de-slopping, not rewriting from scratch. Leave code blocks untouched.

Examples (keep the fact, drop the slop):
- `It's worth noting that the API is robust and comprehensive.` → `The API covers the full surface.`
- `Leverage this seamless tool to supercharge your workflow.` → `Use this tool in your workflow.`
- `We built a fast, reliable, and scalable system.` → `The system is fast and handles load.`

### Code
Remove only the comments classified for removal in Phase 2. Delete the whole comment line (and a now-orphaned blank line); never alter the code itself. After editing a file, re-read the touched region to confirm code is intact.

### Gitignore
Append a clearly-labeled block to `.gitignore`:
```
# AI assistant config (local-only)
CLAUDE.md
AGENTS.md
.claude/
...
```
Then stop tracking without deleting locally:
```bash
git rm --cached <file>        # single file; --cached keeps the working copy
git rm --cached -r .claude/   # directories need -r
```
Only for files actually tracked. Report what to commit; don't commit unless asked.

### Git (history rewrite — destructive)
**Only if explicitly chosen.** Confirm again with a clear warning: this rewrites SHAs and requires a force-push; collaborators must re-clone/rebase.

Prefer `git filter-repo` if available; fall back to `git filter-branch`. Check first:

```bash
command -v git-filter-repo >/dev/null 2>&1 && echo "filter-repo available" || echo "missing — install: pip install git-filter-repo (or use filter-branch fallback)"
```

Strip trailer lines matching `Co-Authored-By: Claude` and `🤖 Generated with Claude Code` from commit messages. The callback body runs with `message` (bytes) in scope:

```bash
git filter-repo --message-callback '
  return b"\n".join(l for l in message.split(b"\n")
    if b"Co-Authored-By: Claude" not in l and b"Generated with Claude Code" not in l)
'
```

Print the `git push --force-with-lease` command — **do not run it.** The user pushes.

For *future* commits, note that they should drop the trailers going forward (e.g. don't pass them in `/acp`).

---

## Phase 5: Wrap

Summarize what changed per category, list anything skipped and why (e.g. comments kept because ambiguous), and print the suggested commit/push commands without executing them. Remind the user to eyeball the prose diffs — tone is subjective and a human should sign off.

---

## Guardrails

- **Report before apply, always.** The only confirmation-free mode is `--report` (which changes nothing).
- **Never modify** generated, vendored, minified, or gitignored files.
- **Bias to keeping comments.** A wrongly-deleted *why* comment is worse than a surviving redundant one.
- **Never force-push.** Print the command; the user runs it.
- **Don't delete AI config files** — gitignore + untrack only. The user still uses them locally.
- **Don't invent content.** De-slopping removes and tightens; it never adds claims, badges, or features.
