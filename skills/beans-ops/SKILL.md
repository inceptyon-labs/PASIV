# Beans Task Operations

Perform beans operation: $ARGUMENTS

**Note**: `beans prime` runs at SessionStart and PreCompact via project hooks, so the agent already has full Beans CLI context. This skill handles PASIV-specific operations only.

## Configuration

Read `.pasiv.yml` for beans settings:
- `path`: directory for beans files (default: `.beans`)
- `prefix`: ID prefix (default: `beans-`)

## Type Mapping

| PASIV Type | Beans Type |
|------------|------------|
| Epic | milestone |
| Feature | feature |
| Task | task |

## Available Operations

### create

Arguments: `"Title" "Body" "labels" "type" [parent]`

```bash
# Map PASIV type to beans type
# Epic → milestone, Feature → feature, Task → task

# Without parent:
beans create "$TITLE" -t $BEANS_TYPE --body "$BODY" --tags "$LABELS"

# With parent:
beans create "$TITLE" -t $BEANS_TYPE --body "$BODY" --tags "$LABELS" --parent $PARENT_ID
```

Return the created bean ID.

### get

Arguments: bean ID

```bash
beans show $ID --json
```

Extract and return: id, title, status, body, tags, type, parent.

### close

Arguments: bean ID, comment

```bash
beans update $ID -s completed
```

If comment provided, append to body.

### get-sub-issues

Arguments: bean ID

```bash
beans query '{ beans(filter: {parent: "'$ID'"}) { id title status } }'
```

Return list of child beans.

### get-parent

Arguments: bean ID

```bash
beans show $ID --json
```

Extract and return parent field. If no parent, return empty.

### check-off-criteria

Arguments: bean ID

```bash
# Get current body
BODY=$(beans show $ID --json | jq -r '.body')

# Replace [ ] with [x]
UPDATED=$(echo "$BODY" | sed 's/- \[ \]/- [x]/g')

# Update
beans update $ID --body "$UPDATED"
```

### add-completion-summary

Arguments: bean ID, files, decisions, notes

```bash
beans update $ID --append-body "## Completed

**Files changed:**
$FILES

**Key decisions:**
$DECISIONS

**Notes for next task:**
$NOTES

---
*Auto-generated by PASIV*"
```

### get-sibling-context

Arguments: bean ID

1. Get this bean's parent ID
2. Query all children of that parent with status=completed
3. For each completed sibling, extract completion summary from body
4. Return collected summaries

```bash
PARENT=$(beans show $ID --json | jq -r '.parent // empty')
if [ -n "$PARENT" ]; then
  beans query '{ beans(filter: {parent: "'$PARENT'", status: "completed"}) { id title body } }'
fi
```

Filter body content for "## Completed" sections and return.
