# Local Markdown Task Operations

Perform local task operation: $ARGUMENTS

## Configuration

Read `.pasiv.yml` for local settings:
- `path`: directory for task files (default: `docs/tasks`)

## File Format

Each task is a markdown file at `{path}/{id}--{slugified-title}.md`:

```markdown
---
id: task-001
title: Create user table
status: open          # open | in-progress | closed
type: task            # epic | feature | task
priority: medium
labels: [area:db, size:XS]
parent: feature-001   # optional
created: 2026-02-16
---

## Description
{body content}

## Acceptance Criteria
- [ ] Criterion 1
- [ ] Criterion 2
```

## ID Generation

IDs follow the pattern `{type}-{sequential-number}`:
- `epic-001`, `epic-002`
- `feature-001`, `feature-002`
- `task-001`, `task-002`

To find the next number, glob existing files and increment:
```bash
LAST=$(ls docs/tasks/{type}-*--*.md 2>/dev/null | sort -V | tail -1 | grep -oP '{type}-\K\d+')
NEXT=$(printf "%03d" $((${LAST:-0} + 1)))
```

## Slugify

Convert title to lowercase, replace spaces and special characters with hyphens, strip trailing hyphens:
```bash
SLUG=$(echo "$TITLE" | tr '[:upper:]' '[:lower:]' | sed 's/[^a-z0-9]/-/g' | sed 's/--*/-/g' | sed 's/-$//')
```

## Available Operations

### create

Arguments: `"Title" "Body" "labels" "type" [parent]`

1. Determine next ID for the given type
2. Slugify the title
3. Write file to `{path}/{id}--{slug}.md` with frontmatter and body
4. Return the ID

### get

Arguments: local ID (e.g., `task-001`)

```bash
# Find file matching ID
FILE=$(ls docs/tasks/${ID}--*.md 2>/dev/null | head -1)
```

Read the file. Parse frontmatter and return: id, title, status, type, body, labels, parent.

### close

Arguments: local ID, comment

1. Read the task file
2. Change `status: open` or `status: in-progress` to `status: closed`
3. If comment provided, append to body
4. Write updated file

### get-sub-issues

Arguments: local ID

Find all files where `parent: {ID}` in frontmatter:

```bash
grep -l "^parent: $ID$" docs/tasks/*.md 2>/dev/null
```

Read each file and return: id, title, status.

### get-parent

Arguments: local ID

Read the task file, extract `parent` from frontmatter. If no parent, return empty.

### check-off-criteria

Arguments: local ID

1. Read the task file
2. Replace `- [ ]` with `- [x]` in the Acceptance Criteria section
3. Write updated file

### add-completion-summary

Arguments: local ID, files, decisions, notes

1. Read the task file
2. Append completion summary section to body:
```markdown

## Completed

**Files changed:**
{files}

**Key decisions:**
{decisions}

**Notes for next task:**
{notes}

---
*Auto-generated by PASIV*
```
3. Write updated file

### get-next
Get the highest priority actionable task.

1. Glob all task files: `docs/tasks/task-*--*.md`
2. Read frontmatter of each, filter to `status: open`
3. Sort by priority: high → medium → low
4. Return the first match: id and title

### get-sibling-context

Arguments: local ID

1. Read task file, get parent ID
2. Find all files with same parent
3. Filter to status: closed
4. Extract "## Completed" sections from each
5. Return collected summaries
